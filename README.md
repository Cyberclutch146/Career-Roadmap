# RoadmapAI

An AI-powered educational platform that generates structured learning roadmaps personalized to your goals, skill level, and available study time.

![RoadmapAI Preview](https://via.placeholder.com/1200x600?text=RoadmapAI+Preview)

## Features

### Core Features

- **AI Roadmap Generator**: Enter your learning goal, current skill level, daily study time, preferred learning style, and target duration. Get a comprehensive, structured roadmap generated by AI.

- **Interactive Learning Chapters**: Navigate through phases, chapters, and lessons. Expand/collapse sections, mark lessons as complete, and track your progress visually.

- **Resource Recommendations**: Each lesson includes curated resources including documentation, videos, articles, courses, GitHub repositories, and practice exercises.

- **AI Mentor Assistant**: Chat with an AI mentor that understands your roadmap and provides contextual guidance. Ask questions like "What should I learn next?" or "Explain recursion in simple terms."

- **Progress Tracking**: Visualize your learning journey with progress bars, completion percentages, and mastery indicators.

- **Study Planner**: Get intelligent scheduling that breaks down your roadmap into daily and weekly actionable tasks.

## Getting Started

### Prerequisites

- Node.js 18+
- Python 3.9+
- MongoDB (local or Atlas)
- Gemini API key (optional, falls back to template roadmaps)

### Installation

#### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/roadmapai.git
cd roadmapai
```

#### 2. Set Up Backend

```bash
cd backend

# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Copy environment variables
cp .env.example .env

# Edit .env with your values:
# - GEMINI_API_KEY (optional)
# - MONGODB_URI (default: mongodb://localhost:27017)
# - JWT_SECRET (change in production!)
```

#### 3. Set Up Frontend

```bash
cd frontend

# Install dependencies
npm install

# Copy environment variables
cp .env.example .env.local

# Edit .env.local if needed
```

### Running the Application

#### Start Backend

```bash
cd backend
uvicorn main:app --reload --port 8000
```

The API will be available at `http://localhost:8000`
API documentation at `http://localhost:8000/docs`

#### Start Frontend

```bash
cd frontend
npm run dev
```

The app will be available at `http://localhost:3000`

### Environment Variables

#### Backend (.env)

```env
# Gemini API Key (get from https://makersuite.google.com/app/apikey)
GEMINI_API_KEY=your_api_key_here

# MongoDB Connection
MONGODB_URI=mongodb://localhost:27017
DATABASE_NAME=roadmapai

# JWT Secret (change this in production!)
JWT_SECRET=your-super-secret-key-change-in-production
```

#### Frontend (.env.local)

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## API Endpoints

### Authentication

- `POST /api/auth/register` - Create new account
- `POST /api/auth/login` - Sign in

### Roadmaps

- `POST /api/roadmaps/generate` - Generate new roadmap
- `GET /api/roadmaps/{id}` - Get roadmap by ID
- `GET /api/roadmaps` - Get user's roadmaps (requires auth)
- `PUT /api/roadmaps/{id}/progress` - Update lesson progress
- `GET /api/roadmaps/{id}/progress` - Get roadmap progress
- `DELETE /api/roadmaps/{id}` - Delete roadmap

### AI Chat

- `POST /api/chat` - Chat with AI mentor
- `GET /api/chat/{roadmap_id}/history` - Get chat history

### Health

- `GET /` - API info
- `GET /health` - Health check

## Roadmap Output Format

Generated roadmaps include:

1. **Overview** - Title, description, total hours, lesson count, difficulty range
2. **Learning Objectives** - Key goals to achieve
3. **Timeline Weeks** - Week-by-week focus areas
4. **Learning Phases** - Structured phases with chapters and lessons
5. **Resources** - Curated resources by category
6. **Revision Strategy** - How to retain learned content
7. **Interview Preparation** - Guidance for technical interviews
8. **Final Assessment** - How to validate mastery

## Deployment

### Deploy Backend to Railway/Render

1. Create new Railway/Render project
2. Connect GitHub repository
3. Set build command: `pip install -r requirements.txt`
4. Set start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. Add environment variables

### Deploy Frontend to Vercel

1. Create new Vercel project
2. Connect GitHub repository
3. Set root directory: `frontend`
4. Add environment variable: `NEXT_PUBLIC_API_URL`

### MongoDB Atlas Setup

1. Create free cluster at mongodb.com/atlas
2. Create database user
3. Whitelist IP `0.0.0.0/0` for development
4. Copy connection string to `MONGODB_URI`
